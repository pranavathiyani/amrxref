AMRxref v0.1 is a sequence-identity crosswalk anchored to the Antibiotic Resistance Ontology (ARO). Every record starts from a CARD entry and adds cross-references to the other databases where the protein or nucleotide sequence is byte-for-byte identical (verified by SHA256 hashing). Below is exactly what shipped, with the gaps stated up front.

Known gaps

AMRFinderPlus integration (v0.2)

v0.1 finds zero CARD ↔ AMRFinderPlus exact-sequence matches even though both databases cover the same genes. The cause is biological: AMRFinderPlus stores mature processed proteins (signal peptide cleaved) while CARD often stores the precursor. Same gene, different bytes, no SHA256 match.

v0.2 will bridge them via NCBI protein accession (RefSeq WP_*), which both databases store and which identifies the same biological entity by definition. This is expected to unlock cross-references for ~6,000 AMRFinderPlus alleles.

Manual gold-tier curation (v0.3+)

All 4,778 records are bronze. Gold-tier curation requires two independent evidence types per mapping (sequence + literature, or sequence + curator endorsement) and human review. Initial gold focus: clinically priority β-lactamases (TEM, SHV, CTX-M, KPC, NDM, OXA), carbapenemases, mobile colistin resistance (mcr), and WHO/CDC priority pathogen AMR genes.

Mycobacterium tuberculosis (sibling project)

MTB resistance is dominated by point mutations in core housekeeping genes (rpoB, katG, pncA, gyrA) and the relevant reference resources are TBProfiler, the WHO Catalogue of mutations, and Mykrobe — not the four databases AMRxref bridges. A partial MTB slice from CARD is in v0.1, but full TB harmonization is a separate project: AMRxref-MTB.

Promoter and regulatory variants (v0.3+)

The schema supports variant_type: promoter and regulatory_region but v0.1 doesn't populate them. Promoter mutations (ampC, mgrB, pmrA/B) confer real resistance via expression changes and currently land in the regulatory_region bucket as a fallback. v0.3 will resolve them properly.

UniProt and AlphaFold cross-links (v0.2)

The schema reserves a canonical_protein.uniprot field but v0.1 doesn't populate it. v0.2 will run NCBI accession ↔ UniProt lookups via the UniProt ID-mapping API, which then unlocks AlphaFold DB links automatically (AFDB is keyed on UniProt).
