Methods

Approach

AMRxref produces canonical cross-references between AMR gene databases by anchoring every record to an ARO term and emitting SKOS mapping relations to entries in CARD, AMRFinderPlus, ResFinder, MEGARes, NCBI, and UniProt.

Pipeline

  1. Pull reference sequences from each source database at pinned versions (data/sources.yaml).
  2. Cluster proteins at 100% identity with MMseqs2 to find equivalent entries.
  3. Cluster nucleotide-only references (rRNA, ResFinder DNA) separately.
  4. Anchor each cluster to an ARO term.
  5. Emit one YAML record per gene with explicit evidence and provenance.
  6. CI builds derived TSV / JSON / JSON-LD / SKOS Turtle on every push.

Mapping relations (SKOS)

RelationMeaning
skos:exactMatchIdentical sequences at the declared level (protein or nucleotide)
skos:closeMatchNear-identical (allele variants, >99% identity)
skos:broadMatchTarget is a parent family of the canonical gene
skos:narrowMatchTarget is a specific allele under a family
skos:relatedMatchMechanistically related but not equivalent

Quality tiers

TierCriteria
GoldManually reviewed with two or more independent evidence types
SilverSingle strong evidence (100% sequence identity verified automatically)
BronzeAutomated only, awaiting human review

Variant types

Each record declares a variant_type:

  • protein_coding — protein-level equivalence; nucleotide carried for ResFinder
  • rrna — equivalence at nucleotide level (no protein)
  • promoter — coordinate-anchored regulatory variant
  • regulatory_region — broader regulatory element

FAIR

  • F Persistent IRIs at https://w3id.org/amrxref/, Zenodo DOI per release
  • A HTTPS, content negotiation (HTML, JSON, JSON-LD, Turtle, TSV)
  • I Anchored to ARO; SKOS for cross-database mappings; aligned with PHA4GE/hAMRonization
  • R CC0 for data, MIT for code; per-mapping provenance; SemVer
